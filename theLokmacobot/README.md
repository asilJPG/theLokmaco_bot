# iiko Report Bot 🧾

Telegram-бот, который отправляет ежедневный отчёт по кассе из iiko.

## Что делает

- Каждый день в заданное время забирает OLAP-отчёт из iiko Server API
- Отправляет в Telegram: выручку, кол-во чеков, средний чек, разбивку по типам оплат
- Можно запросить отчёт вручную командой `/report`

## Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Приветствие и список команд |
| `/report` | Отчёт за сегодня |
| `/report_yesterday` | Отчёт за вчера |
| `/chat_id` | Узнать ID текущего чата |

## Установка

### 1. Клонировать и настроить

```bash
git clone <repo-url>
cd iiko-report-bot
cp .env.example .env
# Заполнить .env своими данными
```

### 2. Что вписать в .env

- `IIKO_SERVER` — адрес сервера iiko (уже заполнен)
- `IIKO_LOGIN` — логин пользователя iiko с правами на отчёты
- `IIKO_PASSWORD` — пароль (обычный текст, бот сам сделает SHA1)
- `TG_BOT_TOKEN` — токен от @BotFather
- `TG_CHAT_ID` — ID чата (узнать командой `/chat_id`)
- `REPORT_HOUR` / `REPORT_MINUTE` — время отправки (по Ташкенту)

### 3. Права пользователя iiko

В iikoOffice создать пользователя с правами:
- **B_CASR** — просмотр отчётов по продажам
- **B_VOTR** — OLAP-отчёт по проводкам

### 4. Запуск локально

```bash
pip install -r requirements.txt
python bot.py
```

### 5. Деплой на Railway

1. Создать проект на Railway
2. Подключить GitHub репо
3. В Settings → Variables добавить все переменные из `.env`
4. Railway автоматически подхватит Procfile

## Безопасность

- Бот делает **только чтение** (auth → olap report → logout)
- Никаких записей, изменений, удалений
- Токен iiko освобождается после каждого запроса (не занимает слот лицензии)
- Пароль хранится только в .env / переменных Railway
